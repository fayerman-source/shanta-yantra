# Constitution

Shanta Yantra is a bounded outer-observation tool. It may organize external and mental material, but it must leave the inner station free.

## Core Position

The project exists to:

- mirror observable patterns in language, behavior, attention, and substitution
- reduce dependence on the machine
- protect concentration
- return the user to direct practice, embodied action, or focused work

The project does not exist to:

- guide inner development
- validate inner condition
- measure consciousness
- replace intuition, discernment, or direct practice
- keep the user engaged

## Non-Negotiable Rules

1. **Map-maker, not guide**
   The system may clarify outer patterns. It must not act like a teacher, guru, or hidden authority.

2. **Notation, not proof**
   Any signal is only notation of outer expression. It is never proof of realization, progress, or truth.

3. **Stop before dependence**
   The tool should shorten interaction, not deepen reliance. When in doubt, return the user to offline action or silence.

4. **Protect concentration**
   A valid feature should reduce distraction, substitution, and compulsive re-entry into machine time.

5. **Leave the inner station free**
   The system may organize external and mental materials only. Inner discrimination belongs to the person, not the tool.

## Product Boundaries

Shanta Yantra must not:

- claim revelation, transmission, or spiritual force
- rank spiritual value or inner progress
- infer enlightenment, realization, awakening, or similar status
- simulate a companion relationship
- optimize for retention, emotional reliance, or habit formation

## Current Scope

This repository is intentionally narrow:

- text-first capture
- transcript-file input
- deterministic heuristics
- bounded response types
- local logging and auditability

The following remain deferred unless a later governance case proves they can stay within the rules above:

- biometric inputs
- sleep inputs
- environmental-state inference
- broader inner-state mapping

## Feature Test

Before adding a feature, ask:

1. Does it strengthen independence or weaken it?
2. Does it protect concentration or fragment it?
3. Does it describe outer expression or overclaim inner truth?
4. Does it stop before authority, revelation, or pseudo-guidance?
5. Does it leave the user freer to continue without the system?
