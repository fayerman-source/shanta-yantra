# Security Policy

## Supported Versions

Security fixes are currently expected on the latest `0.1.x` line on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability that could expose user data, create unsafe behavior, or materially weaken safety-related boundaries.

Use one of these channels instead:

- GitHub Security Advisories for this repository, if enabled
- direct maintainer contact through the repository profile

When reporting, include:

- a short description of the issue
- affected files or commands
- steps to reproduce
- expected impact
- any suggested mitigation

You will receive an acknowledgment as soon as practical, followed by triage and next-step guidance.

## Scope

For this repository, security includes both traditional software concerns and failures that could bypass explicit safety redirects or boundary constraints in the CLI behavior.
