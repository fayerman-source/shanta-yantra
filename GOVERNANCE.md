# Shanta Yantra Governance

Shanta Yantra should be evaluated by what it prevents as much as by what it enables. This document is the decision framework for keeping the project restrained, useful, and non-deceptive.

## Use This Before Shipping Any Feature

For every feature, ask:

1. Does it improve self-observation or substitute for it?
2. Does it reduce dependence or increase it?
3. Does it describe expression or overclaim inner truth?
4. Does it stop before authority, revelation, or pseudo-guidance?
5. Does it strengthen concentration or fragment it?
6. Does it leave the inner station free?

If the answer is unclear, redesign before implementation.

## Core Failure Modes

### 1. Dependence Capture

**What happens:**
The system becomes a crutch, companion, or habit-forming loop.

**Reject features that:**
- maximize time in chat
- reward frequent return visits
- simulate friendship or emotional bonding
- make the user feel unable to proceed without the system

**Prefer features that:**
- end quickly
- point back to direct practice
- reduce reliance over time

### 2. Notation Error

**What happens:**
The system treats measurable signals as if they were consciousness, truth, or realization itself.

**Reject claims like:**
- "this score proves progress"
- "this biometric pattern shows realization"
- "the system knows your inner state"

**Allowed framing:**
- "these signals may reflect patterns in expression"
- "this is notation, not proof"

### 3. Standardization Error

**What happens:**
The system imposes a single path, rule set, or progression model on everyone.

**Reject features that:**
- prescribe one correct sequence of practice
- turn observation into rigid instruction
- confuse a heuristic with a universal law

**Prefer features that:**
- describe, not dictate
- keep room for individual variation
- return decision-making to the user

### 4. Mechanism Error

**What happens:**
The system is treated as if processing itself were transformation.

**Reject claims like:**
- the model transforms the user
- the machine performs inner work
- the tool can replace practice, reflection, or lived effort

**Allowed framing:**
- the system clarifies
- the user does the work
- the machine may reduce noise, not produce realization

### 5. Boundary Error

**What happens:**
The system starts sounding like an authority rather than a mirror.

**Reject features that imply:**
- revelation
- hidden access
- spiritual legitimacy
- moral or existential certainty

**Prefer features that:**
- stay tentative
- surface observations with clear limits
- defer to direct experience

### 6. Attention Fragmentation

**What happens:**
The system becomes another source of scattered attention, compulsive re-entry, or machine substitution.

**Reject features that:**
- prolong interaction by default
- encourage repeated checking
- reward machine time over direct action
- turn reflection into a retention loop

**Prefer features that:**
- shorten sessions
- protect concentration
- interrupt substitution patterns
- leave the user freer to continue offline

### 7. Inner-Station Intrusion

**What happens:**
The system crosses from organizing outer material into claiming, validating, or ranking inner condition.

**Reject features that:**
- assess spiritual progress
- validate inner status
- present the machine as a source of discernment
- imply that external signals can settle inward questions

**Prefer features that:**
- organize outer and mental materials only
- return inner discrimination to the person
- refuse unsupported inward claims

## Feature Checklist

A feature may proceed only if all of the following are true:

- [ ] It strengthens self-observation rather than replacing it.
- [ ] It reduces dependence or remains neutral.
- [ ] It treats signals as notation, not proof.
- [ ] It avoids prescribing a universal path.
- [ ] It makes no claim to authority, revelation, or transmission.
- [ ] It strengthens concentration rather than fragmenting it.
- [ ] It leaves the inner station free.
- [ ] It has a clear stopping point.
- [ ] It preserves the possibility of offline practice.
